# ampsw Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Bun + TypeScript CLI named `ampsw` that saves and switches Amp accounts on macOS and Linux and ships through curl, npm, and Homebrew installation paths.

**Architecture:** The CLI will be split into focused command modules, core account-switching logic, secure storage adapters, and packaging scripts. Runtime logic will only depend on Amp's `secrets.json`, while distribution logic will produce Bun-compiled executables and thin packaging layers for npm and Homebrew.

**Tech Stack:** Bun, TypeScript, Bun test, GitHub Actions, shell install script, npm packaging metadata, Homebrew tap formula generation.
